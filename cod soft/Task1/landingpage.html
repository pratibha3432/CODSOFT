ABOUT US

An SP stack provides a given combination of support package levels of different component versions. You should only deviate from this combination in exceptional cases. An SP stack may contain software components that are not used productively. They don't need to be patched during the implementation of the SP stack unless a technical or logical dependency to the used components exists.
